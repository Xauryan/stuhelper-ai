# 自助充值

自助充值允许管理员在系统设置中上传微信和支付宝个人收款码。用户充值或购买订阅时选择“微信自助”或“支付宝自助”，扫码付款后手动提交充值金额和微信/支付宝交易订单号。额度充值会先实时为用户增加余额；订阅购买会先实时开通订阅；随后由管理员在账单管理中每日对账审核。

## 配置入口

入口：`系统设置` -> `支付设置` -> `自助充值`。

配置项：

- `SelfServeTopUpEnabled`：自助充值总开关。
- `SelfServeAlipayEnabled`：是否启用支付宝自助充值。
- `SelfServeWechatPayEnabled`：是否启用微信自助充值。
- `SelfServeAlipayQRCode`：支付宝收款码内容，通常由前端上传图片后解码写入，也可手动粘贴二维码内容或支付链接。
- `SelfServeWechatPayQRCode`：微信收款码内容，通常由前端上传图片后解码写入，也可手动粘贴二维码内容或支付链接。
- `SelfServeTopUpUnitPrice`：自助充值独立价格，表示 `1` 美金额度对应多少人民币元。该配置只用于自助充值，不使用易支付的 `Price`。
- `SelfServeTopUpSingleMaxAmount`：每人单笔自助充值金额上限，单位为人民币元。
- `SelfServeTopUpDailyMaxAmount`：每人每日自助充值累计金额上限，单位为人民币元。

前端展示二维码时统一使用保存的二维码内容重新生成二维码。只有 `data:image/...` 或明确以 `png`、`jpg`、`jpeg`、`webp`、`gif` 结尾的历史图片 URL 才按图片渲染；`https://qr.alipay.com/...` 等支付链接必须作为二维码内容渲染，不能直接作为 `<img src>` 使用。

二维码限制：

- 前端上传限制为 `PNG/JPG/WebP` 且不超过 `300KB`。
- 前端会先解码上传图片，只保存二维码文本内容；用户充值或购买订阅时再由前端重新生成二维码展示。
- 历史已保存的 `http/https` 图片地址或 `data:image/...;base64,...` 仍会兼容展示，但新配置不再建议保存原图。
- 后端配置校验允许空值、普通二维码文本/支付链接和历史图片值；普通文本最多 4096 个字符，历史 `data:image/png|jpeg|jpg|webp;base64,...` 最大 `512KB`。

## 风控限额

自助充值价格和限额都是独立配置项。管理员必须手动确认 `SelfServeTopUpUnitPrice`，并配置 `SelfServeTopUpSingleMaxAmount` 和 `SelfServeTopUpDailyMaxAmount` 后，用户才可以使用自助充值；每日限额必须大于或等于单笔限额。`199.99`、`299.99` 等运营限额不写死在代码中，必须由管理员在系统设置中手动配置。

用户充值弹窗、订阅自助支付弹窗、管理员编辑弹窗和后端校验都会使用当前配置值展示和限制金额。未完整配置自助价格或限额时，即使总开关和收款码已配置，前端也不会展示自助充值入口，后端也会拒绝预览、提交、订阅购买和编辑请求。

每日限额按用户、自然日、本地服务端时区统计，包含待审核和已通过的自助充值记录，也包含待审核的自助订阅购买记录；已拒绝记录不计入每日已用额度。后端会在预览、提交、订阅购买和管理员编辑时强制校验，前端限制仅作为交互提示。

## 用户流程

1. 用户进入充值页，选择“支付宝自助”或“微信自助”。
2. 系统展示对应收款码、单笔/每日限额、今日剩余额度和风险提示。
3. 用户扫码付款后填写：
   - 充值金额，单位为人民币元。
   - 交易订单号，必须填写微信或支付宝账单里的交易订单号，不是商户订单号。
4. 用户确认“已完成付款，并承诺金额和交易订单号真实有效”后提交。
5. 系统立即创建一笔成功充值账单和一条待审核记录，并实时增加用户余额。

风险提示必须保留：

- 提交后余额会实时到账。
- 虚假填写、重复提交或金额不符会被拒绝、扣回余额，账户可能被封禁。
- 自助充值拒绝后概不退款。

## 到账额度计算

用户填写的是人民币充值金额。系统按自助充值独立价格和用户充值分组倍率换算到账额度：

```text
到账额度 = 充值金额 / SelfServeTopUpUnitPrice / TopupGroupRatio(user.group) * QuotaPerUnit
```

结果向下取整。若系统价格、额度倍率或充值分组倍率配置错误，后端会拒绝提交。

充值页选择“支付宝自助”或“微信自助”时，自动填写的扫码付款金额也按 `SelfServeTopUpUnitPrice` 和当前用户的 `TopupGroupRatio(user.group)` 反推，不复用易支付 `Price`、易支付手续费或其他支付方式遗留的实付金额状态。

## 自助购买订阅

自助支付也可以用于购买订阅套餐。用户在订阅套餐页选择“支付宝自助”或“微信自助”后，系统会按套餐 `price_amount * SelfServeTopUpUnitPrice` 向上保留两位小数计算应付人民币金额，并展示对应收款码。

提交订阅自助支付时：

- 请求接口为 `POST /api/subscription/self-serve/pay`。
- 用户必须填写真实微信/支付宝交易订单号。
- 后端要求申报金额严格等于系统计算的应付金额。
- 成功后立即创建成功订阅订单、同步账单记录、创建待审核自助审核记录，并开通订阅。
- 订阅自助支付不扣用户钱包余额，不使用易支付价格和手续费。

管理员审核订阅自助支付时，仍在账单管理中处理同一条自助审核记录。通过后会发放订阅返佣；拒绝后会把订阅订单和对应账单标记为已退款/已拒绝，并取消订阅、按订阅分组规则回退用户分组。

## 管理员审核

入口：`账单管理`。

账单管理新增：

- 支付方式筛选：`自助充值`。
- 账单视图：`待审核自助充值`。
- 列字段：交易订单号、申报金额、审核状态。
- 待审核记录操作：
  - `通过`：确认真实到账后将审核状态改为 `approved`，并在通过时发放邀请返佣。
  - `编辑`：纠正申报金额、交易订单号和审核备注。保存后系统会按新金额重新计算到账额度，并按差额调整用户余额。
  - `拒绝`：将审核状态改为 `rejected`，扣回该订单已到账余额或取消自助订阅，创建退款/扣回记录，并把充值账单标记为已退款。

系统设置中不再提供“拒绝时默认封禁”开关。账单管理只展示一个“拒绝”按钮；管理员需要封禁时，在拒绝确认弹窗中勾选“同时封禁该用户”，系统会在拒绝扣回的同时禁用用户账户并失效用户缓存/令牌缓存。

## 自助退款

自助充值和自助订阅没有可直接调用的第三方退款 API。普通用户在自己的账单中对自助支付成功或部分退款订单申请退款时，除退款金额和退款原因外，还必须上传或粘贴退款收款码：

- 上传图片时，前端会解码二维码，只把二维码内容提交给后端。
- 后端把二维码内容保存到 `top_up_refund_requests.refund_qrcode`，不会保存上传原图。
- 管理员在账单管理审批退款时可以查看二维码预览和可复制文本，手动完成退款后再点击“审批通过并退款”登记系统。
- 自助退款可退金额复用官方支付退款预览逻辑：额度充值按未使用余额折算，订阅按未使用时间/额度比例折算，手续费不参与退款。

管理员审批自助退款时不会调用支付宝或微信退款接口，只会登记一条本地成功退款记录。额度充值退款会扣回用户余额并冲回返佣；订阅退款会同步订阅订单状态，部分退款保留订阅，满足全额退款条件时取消订阅并回退分组。拒绝退款申请只更新申请状态，不扣余额、不调用外部接口。

## API

用户接口：

- `POST /api/user/self-serve/preview`
  - 请求：`declared_money`
  - 返回：预计到账额度、今日已用金额、今日剩余金额、当前配置限额。
- `POST /api/user/self-serve/pay`
  - 请求：`payment_method`、`declared_money`、`transaction_no`
  - 支付方式：`alipay_self_serve`、`wxpay_self_serve`
  - 成功后立即入账，并生成待审核记录。
- `POST /api/subscription/self-serve/pay`
  - 请求：`plan_id`、`payment_method`、`declared_money`、`transaction_no`
  - 支付方式：`alipay_self_serve`、`wxpay_self_serve`
  - 成功后立即开通订阅，并生成待审核记录。
- `POST /api/user/topup/official/refund/apply`
  - 官方支付退款申请沿用该接口；自助支付订单也走同一接口，但必须额外提交 `refund_qrcode`。

管理员接口：

- `POST /api/user/topup/self-serve/approve`
  - 请求：`trade_no`、`reason`
- `POST /api/user/topup/self-serve/update`
  - 请求：`trade_no`、`declared_money`、`transaction_no`、`reason`
- `POST /api/user/topup/self-serve/reject`
  - 请求：`trade_no`、`reason`、`ban_user`

账单查询：

- `payment_method=self_serve`：筛选自助充值账单。
- `audit_status=pending|approved|rejected`：筛选自助充值审核状态。
- `pending_refund=true`：筛选待审批退款申请，自助支付退款申请也会出现在该视图。

## 数据表

自助充值审核记录存储在 `self_serve_top_up_audits`：

- `top_up_id`：关联充值账单。
- `trade_no`：系统充值订单号。
- `transaction_no`：用户填写的微信/支付宝交易订单号，全局唯一。
- `payment_method`：`alipay_self_serve` 或 `wxpay_self_serve`。
- `declared_money`：用户申报金额。
- `credited_quota`：本次已实时到账额度。
- `status`：`pending`、`approved`、`rejected`。
- `admin_reason`、`auditor_id`、`reviewed_time`：管理员审核信息。

自助退款申请复用 `top_up_refund_requests`：

- `refund_qrcode`：用户提交的退款收款码内容，只保存解码后的文本/链接。

## 运营建议

- 管理员应每天至少核对一次待审核自助充值。
- 审核时以微信/支付宝实际到账记录为准，核对交易订单号和金额。
- 对金额填错但确实到账的记录，优先使用“编辑”纠正后再通过。
- 对未到账、重复提交、恶意虚假提交的记录，使用“拒绝”；需要封禁时，在拒绝确认弹窗中勾选“同时封禁该用户”。
